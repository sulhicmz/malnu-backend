# Security Policy

## Supported Versions

We release security updates for the following versions of our application:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | ✅ Latest          |
| < 1.0   | ❌ Not supported   |

## Reporting a Vulnerability

We take the security of our application seriously. If you discover a security vulnerability, please report it to us responsibly.

### How to Report

**Do not** create a public GitHub issue for security vulnerabilities. Instead, please report them through one of these channels:

- Email: [security-contact@your-organization.com](mailto:security-contact@your-organization.com) (replace with actual security contact)
- GitHub's private vulnerability reporting feature (use the "Security" tab in the repository)

### What to Include

When reporting a vulnerability, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Your contact information for follow-up
- If possible, a suggested fix or mitigation

### Response Timeline

We commit to:
- Acknowledging your report within 48 hours (business days)
- Providing an initial assessment within 5 business days
- Regular updates on the status of your report
- Notification when the vulnerability is fixed

## Security Update Policy

- Critical security patches will be released as soon as possible
- Security updates are typically released on a monthly schedule
- Users are strongly encouraged to keep their installations up to date

## Preferred Languages

We prefer all communications to be in English, but we also accept reports in Indonesian.

## Additional Security Measures

### Code Review Process
- All code changes undergo security review
- Critical security-related code requires multiple reviewers
- Automated security scanning is performed on all pull requests

### Dependencies
- We regularly update and audit our dependencies
- Security advisories are monitored for all dependencies
- Outdated dependencies with known vulnerabilities are updated promptly

## Recognition

We believe in recognizing the security research community for their valuable contributions. If your vulnerability report leads to a fix, you may be acknowledged in our release notes (unless you prefer to remain anonymous).

## Questions

If you have any questions about this security policy, please contact us at [security-contact@your-organization.com](mailto:security-contact@your-organization.com).
